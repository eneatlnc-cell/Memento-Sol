## 1. 首次启动

首次打开 Memento 需要下载两个模型文件（共约 700MB）：

| 文件 | 大小 | 作用 |
|------|------|------|
| `mmproj-Qwen_Qwen3.5-0.8B-bf16.gguf` | ~200MB | 视觉编码器（处理图片/视频帧） |
| `Qwen3.5-0.8B-UD-Q4_K_XL.gguf` | ~500MB | 主语言模型（文字推理 + SVG 生成） |

**下载流程**：

1. 进入引导页，点击「开始下载」。
2. 应用会先下载视觉编码器，再下载主模型，下载完成后自动加载引擎。
3. 下载支持断点续传，若中断可重新进入应用继续。
4. 引擎初始化需要数秒（JNI 加载），请勿在此期间杀掉应用。
5. 加载完成后进入主聊天界面，即可开始对话。

> ⚠️ 下载需要稳定的网络环境（模型文件托管在阿里云 OSS）。建议在 Wi-Fi 下进行。

## 2. 基础对话

进入主界面后，底部输入栏可直接输入文字与 Memento 对话。

- **发送**：点击右侧发送按钮。
- **停止**：推理过程中发送按钮会变为停止按钮，点击可中断当前生成。
- **流式输出**：Memento 会逐字输出回复，无需等待整段完成。
- **主动搭话**：长时间未交互后回到应用，Memento 会主动问候。
- **记忆**：你的对话内容会以摘要形式存入本地记忆库，用于保持上下文连续性。可在设置中清除。

## 3. 多模态交互

Memento 是原生多模态架构，图片、视频、文字在输入端地位平等，均可作为生成的参考素材。所有处理均在端侧完成。

### 3.0 架构原理：理解 → 翻译 → 渲染

Memento 的多模态生成采用独特的**「理解 → 翻译 → 渲染」**架构，与传统像素直接生成模型有本质区别。

```
输入（自然语言 / 图片 / 视频）
    ↓
多模态理解（Qwen3.5-VL 看懂你的输入）
    ↓
结构化翻译（模型输出 SVG 代码，精确描述画面元素、布局、色彩）
    ↓
渲染管线渲染（WebView 精准渲染 SVG 为 Bitmap / MP4）
    ↓
输出（图片 / 视频）
```

#### 为什么这样设计

传统大模型直接生成像素，写实度高但**无法保证一致性**——同一角色在不同画面中可能长相不同，同一场景重复生成会变形。

Memento 的架构通过结构化描述（SVG）+ 确定性渲染管线，实现：

| 优势 | 说明 |
|------|------|
| **一致性可控** | 同一角色/场景在多次生成中保持一致，适合连续叙事 |
| **结构可编辑** | 生成结果可精确调整（如「把背景换成夜晚」「角色位置左移」） |
| **风格稳定** | 风格由 SVG 描述精确控制，不会出现随机漂移 |
| **多模态输入** | 输入不限于文字，可用图片/视频作为参考素材 |

#### 能力边界（诚实说明）

Memento 使用的是 **Qwen3.5-VL-0.8B** 轻量级模型，其 SVG 输出能力有物理边界：

- ✅ **擅长**：矢量风格图形、几何构图、色块组合、渐变背景、简单场景、图标 Logo、扁平插画
- ⚠️ **勉强**：中等复杂度场景（房子+树+人物）、卡通人物
- ❌ **不擅长**：照片级写实、精细人物面部、复杂纹理、透视场景

> 💡 Memento 的定位是**矢量风格图形/视频生成**，强项是**一致性**与**可编辑性**，不是照片级写实。连续叙事（如漫剧、系列海报）场景下，本架构远优于像素生成模型。

### 3.1 三段式工作流：帧 / 图形 / 合成

Memento 的多模态输入采用独特的**三段式工作流**，通过「+」菜单进入：

```
帧   ─→ 1s 视频 → 24 帧关键帧 → 保存相册 + 进缓存
  │
  ↓ 关键帧进入"图形"工坊
图形 ─→ 元素替换对话框（用户元素+原图元素+关键帧）
  │     用户指令改造特定元素（替换/调整/添加）
  │     输出：批量修改后的关键帧(SVG) → 保存进缓存
  ↓ 改造后的关键帧进入"合成"
合成 ─→ 取当前缓存的关键帧序列(SVG)
       逐帧渲染 → MediaCodec → MP4
       输出：MP4 视频
```

**三段式通过 KeyFrameStore 缓存传递关键帧状态**，让用户可以分步骤完成复杂任务，而不是一次性把所有素材塞给模型。

#### 加号菜单

点击输入栏左侧的 **「+」** 按钮，弹出三个选项：

| 选项 | 图标 | 功能 | 前置条件 |
|------|------|------|---------|
| **帧** | 闪光 | 选择 1s 视频 → 提取 24 帧关键帧 → 保存相册 + 进缓存 | 无 |
| **图形** | 图片 | 元素替换对话框，改造关键帧输出 SVG | 有关键帧缓存时体验最佳 |
| **合成** | 胶片 | 将缓存关键帧合成为 MP4 视频 | 缓存必须有关键帧 |

> 💡 输入栏上方会显示关键帧缓存指示器：`🎬 关键帧缓存：N 帧（来源）`，可一键清空。

### 3.2 帧：1s 视频 → 24 帧关键帧

点击「+」→ **帧**，打开**「帧工坊：1s 视频 → 24 帧关键帧」**对话框。

#### 功能

- **选择 1s 视频**：从系统选择器挑选一个视频（≤1 秒，≤50MB）。
- **提取 24 帧**：系统使用 `MediaMetadataRetriever` 提取 24 帧关键帧（256×256 缩略图）。
- **保存到相册**：提取的帧自动保存到相册的 `Pictures/Memento` 目录。
- **进入缓存**：帧 Uri 自动进入 KeyFrameStore，供「图形」「合成」使用。

#### 限制

| 项目 | 限制 |
|------|------|
| 视频时长 | **≤ 1 秒**（超过部分不采样） |
| 视频大小 | **≤ 50 MB** |
| 采样帧率 | **24 fps**（1 秒 × 24 帧 = 24 帧缩略图） |
| 帧尺寸 | **256×256 缩略图** |
| 超限处理 | 弹出提示框，附件不会被添加 |

#### 操作步骤

1. 点击「+」→ **帧**，打开帧工坊对话框。
2. 点击「选择 1s 视频」，从系统选择器挑选视频。
3. 系统校验大小（≤50MB），超限弹出提示。
4. 提取进度实时显示：「正在提取关键帧... N/24」。
5. 提取完成后，帧保存到相册并进入缓存。
6. 关闭对话框，可在「图形」中改造，或在「合成」中生成 MP4。

#### 为什么是 1 秒 24 帧

- **1 秒**：端侧模型的 token 上限决定了无法处理过长视频，1 秒是平衡点。
- **24 帧**：达到视频最低帧率标准，模型可理解连续动作。
- **256×256 缩略图**：减少 token 消耗，让模型聚焦于内容理解而非细节。

### 3.3 图形：元素替换改造关键帧

点击「+」→ **图形**，打开**「图形工坊：元素替换改造关键帧」**对话框。

#### 对话框布局

```
┌─────────────────────────────────────────┐
│  图形工坊：元素替换改造关键帧           │
├─────────────────────────────────────────┤
│  [+ 用户元素] [+ 原图元素] → [+ 关键帧] │
│    缩略图       缩略图       缩略图×N   │
│                                         │
│  已选关键帧 N 张（来自帧工坊缓存或手动选择）│
│  [缩略图列表，可单独删除]               │
│                                         │
│  选择用户元素和原图元素，系统将替换关键  │
│  帧中的对应元素，输出改造后的 SVG 关键  │
│  帧进入缓存，供「合成」生成 MP4。       │
├─────────────────────────────────────────┤
│              [取消]  [提交]             │
└─────────────────────────────────────────┘
```

#### 三个选择入口

| 入口 | 数量 | 用途 |
|------|------|------|
| 用户元素 | 1 张 | 要替换进去的素材 |
| 原图元素 | 1 张 | 原图中要被替换的部分 |
| 关键帧 | 4-12 张 | 动画基础，自动从 KeyFrameStore 预填 |

#### 关键帧自动预填

打开图形工坊时，若 KeyFrameStore 中已有缓存的关键帧（来自帧工坊），会自动填入「关键帧」入口，无需重复选择。

#### 提交后发生什么

1. 三类素材合并为图片列表，显示为输入框上方的 chip。
2. **若有关键帧**：同步到 KeyFrameStore，供「合成」使用。
3. 在输入框输入说明文字（可选），点击发送。
4. 模型按 caption 标记 `[用户元素] [原图元素] [关键帧×N]` 处理，输出改造后的 SVG 帧序列进入缓存。

#### 模型如何理解你的输入

| 输入组合 | 模型行为 |
|---------|---------|
| 仅用户元素 + 原图元素 | 把原图元素替换为用户元素，输出图片 |
| 仅关键帧 | 理解关键帧序列，输出视频动画 |
| 三者都有 | 把原图元素替换为用户元素，并基于关键帧生成改造后的 SVG 动画 |
| 仅用户元素 或 仅原图元素 | 作为参考图理解，配合文字描述处理 |

#### 使用场景

- **元素替换**：「把原图中的猫替换成我的猫」——用户元素=我的猫，原图元素=原图中的猫。
- **批量改造关键帧**：基于帧工坊提取的 24 帧，把某一元素替换为用户元素，生成改造后的 SVG 序列。
- **纯参考图**：只选一张用户元素作为风格参考，配合文字描述生成图片。

### 3.4 合成：关键帧 → MP4

点击「+」→ **合成**，触发视频合成流程。

#### 前置条件

- KeyFrameStore 中必须有关键帧（否则菜单项禁用并提示）。

#### 合成流程

1. 应用取 KeyFrameStore 中所有关键帧（SVG 或图片 Uri）。
2. 关键帧作为图片序列送入模型。
3. 模型理解关键帧内容与时序，输出 `[GEN_VIDEO]` 标记 + 每帧 SVG。
4. StructuredRenderer 逐帧渲染 SVG → Bitmap。
5. BitmapToVideoEncoder 将 Bitmap 序列编码为 MP4（H.264 + 24fps）。
6. 渲染进度实时显示在对话气泡中。

#### 输出规格

| 项目 | 规格 |
|------|------|
| 输出尺寸 | 512 × 512 像素 |
| 输出格式 | MP4（H.264 编码） |
| 帧率 | 24 fps |
| 帧数 | 等于缓存关键帧数（4-12 帧） |
| 视频时长 | 帧数 / 24fps（如 8 帧 ≈ 0.33 秒） |

> 💡 合成是整个三段式工作流的终点。帧工坊提取关键帧，图形工坊改造关键帧，合成产出 MP4。三段可灵活组合，也可单独使用。

### 3.5 视频输入（1 秒 24 帧）

Memento 支持发送单个短视频，系统会采样视频关键帧送入模型。视频提取的帧也会自动进入 KeyFrameStore，可在「帧」「合成」中继续使用。

#### 限制

| 项目 | 限制 |
|------|------|
| 视频数量 | **单个**（不支持多选） |
| 视频大小上限 | **50 MB** |
| 视频时长 | **前 1 秒**（超过部分不采样） |
| 采样帧率 | **24 fps**（1 秒 × 24 帧 = 24 帧缩略图） |
| 帧尺寸 | **256×256 缩略图**（减少模型 token 负担） |
| 超限处理 | 弹出提示框，附件不会被添加 |

#### 操作步骤

1. 点击输入栏 **「+」** → 选择 **「视频」**。
2. 在系统选择器中挑选一个视频。
3. 选完后返回聊天界面，视频以单 chip 形式显示在输入框上方。
4. **可选**：输入说明文字（如「这是我家狗学接飞盘的过程」）。
5. 点击发送，系统会：
   - 校验视频大小；
   - 使用 `MediaMetadataRetriever` 提取前 1 秒、24 帧关键帧；
   - 将每帧压缩为 256×256 缩略图 JPEG；
   - 作为图片序列送入模型。

#### 为什么是 1 秒 24 帧

- **1 秒**：端侧模型的 token 上限决定了无法处理过长视频，1 秒是平衡点。
- **24 帧**：达到视频最低帧率标准，模型可理解连续动作。
- **256×256 缩略图**：减少 token 消耗，让模型聚焦于内容理解而非细节。

> 💡 想要更精确地控制模型看到的画面？推荐使用「图形」工坊手动挑选关键帧，效果往往优于系统自动采样。

### 3.6 图片输出（SVG 渲染）

Memento 的图片生成基于「理解 → 翻译 → 渲染」架构，模型输出 SVG 代码，渲染管线精准渲染。

#### 核心机制

```
输入（自然语言 / 图片 / 视频）
    ↓
模型理解 + 翻译为 SVG（结构化描述）
    ↓
StructuredRenderer 渲染 SVG → Bitmap（512×512）
    ↓
保存为 PNG 图片
```

#### 输出规格

| 项目 | 规格 |
|------|------|
| 输出尺寸 | **512 × 512** 像素 |
| 输出格式 | PNG |
| SVG 元素上限 | 单张 ≤ 30 个元素 |
| 支持的 SVG 元素 | rect / circle / ellipse / path / polygon / line / linearGradient / radialGradient / text |

#### 触发方式

直接在对话中表达需求即可，输入形式灵活：

- **纯文字**：「画一只戴帽子的猫」
- **文字 + 参考图**：先发送一张参考图，再说「按这个风格画一张生日贺卡」
- **多图参考**：发送多张产品图，说「为这个产品设计一张海报」

#### 风格指引

| 风格 | 适用场景 | 示例描述 |
|------|---------|---------|
| 矢量写实 | 风景、静物、场景概念图 | 「画一幅日落海景，用色块表现」 |
| 扁平插画 | 贺卡、海报、故事插图 | 「画一张温暖风格的生日贺卡」 |
| 极简矢量 | 图标、Logo、几何图形 | 「设计一个极简风格的飞机图标」 |
| 渐变科技 | 科技感、夜景、霓虹 | 「画一个赛博朋克风格的城市夜景」 |

> 💡 描述时强调**矢量风格**效果最佳。Memento 不擅长照片级写实，但矢量风格的一致性和可编辑性是核心优势。

### 3.7 视频输出（逐帧渲染）

Memento 的视频生成基于「理解 → 翻译 → 渲染」架构，模型输出多帧 SVG，渲染管线逐帧渲染后合成为 MP4。

#### 核心机制

```
输入（自然语言 / 图片 / 视频）
    ↓
模型理解 + 翻译为 SVG 帧序列（4-12 帧为宜）
    ↓
StructuredRenderer 逐帧渲染 SVG → Bitmap
    ↓
MediaCodec H.264 编码 → MP4（24fps）
```

#### 输出规格

| 项目 | 规格 |
|------|------|
| 输出尺寸 | **512 × 512** 像素 |
| 输出格式 | MP4（H.264 编码） |
| 帧率 | **24 fps** |
| 帧数 | 模型自由决定（4-12 帧为宜，最多 24 帧） |
| 视频时长 | 帧数 / 24fps（如 8 帧 ≈ 0.33 秒） |
| 编码器 | 优先硬件编码，不支持时回退软件编码 |

#### 触发方式

- **纯文字**：「帮我做一个生日快乐的动画」
- **文字 + 参考图**：发送一张 Logo，说「为这个 Logo 做一个加载动画」
- **文字 + 参考视频**：发送一段 1s 视频，说「按这个风格做一段动画」
- **三段式合成**：通过「图形→帧→合成」工作流产出（见 3.1-3.4）

#### 渲染过程

1. 模型输出 `[GEN_VIDEO]` 标记 + 多个 `<frame>` SVG
2. 应用解析每一帧 SVG
3. StructuredRenderer 逐帧渲染：
   - 每帧 SVG → WebView 加载 → 截图为 Bitmap
   - 渲染失败时用黑帧填充（保证帧数与时序一致）
4. BitmapToVideoEncoder 将 Bitmap 序列编码为 MP4（H.264 + MediaMuxer）
5. 渲染进度实时显示在对话气泡中（如「正在渲染视频（8 帧）... 50%」）

#### 适用场景

- 节日贺卡与祝福动画
- 品牌加载动画与过渡动效
- 短主题动画与分镜演示
- 社交媒体短视频素材

> 💡 视频生成是逐帧渲染过程，帧数越多耗时越久。模型会根据动画复杂度自由决定帧数，简单动画 4-6 帧，复杂动画 8-12 帧。

### 3.8 图片编辑（结构化编辑）

这是「理解 → 翻译 → 渲染」架构的核心优势——**结构可编辑**。

#### 工作原理

当你对已生成的图片提出修改要求时，应用会：

1. 自动缓存最近一次生成的 SVG
2. 检测到编辑意图（如「修改」「替换」「去掉」「换成」）
3. 将原 SVG + 你的编辑指令一起送给模型
4. 模型输出修改后的完整 SVG
5. StructuredRenderer 重新渲染

#### 操作示例

```
你：画一只橙色的猫
Memento：[生成了一张橙色猫的图片]

你：把猫换成黑色
Memento：[基于原 SVG 修改，生成黑色猫的图片]

你：去掉背景，换成夜晚
Memento：[基于原 SVG 修改，生成夜晚背景的黑色猫]

你：猫的位置往左移一点
Memento：[基于原 SVG 修改，生成左移后的图片]
```

#### 支持的编辑指令

| 编辑类型 | 示例 |
|---------|------|
| 颜色替换 | 「把猫换成黑色」「背景改成蓝色」 |
| 元素删除 | 「去掉太阳」「删除背景里的树」 |
| 位置调整 | 「猫往左移」「太阳放高一点」 |
| 大小调整 | 「猫放大一点」「太阳缩小」 |
| 元素添加 | 「加一个月亮」「天上加几朵云」 |

> 💡 编辑模式会自动激活（无需特殊指令），只要检测到编辑关键词且有最近生成的图片。编辑基于 SVG 结构化描述，比像素级修改精确得多。

## 4. Skill 市场

Memento 采用「核心 + Skill」的模块化架构：

- **核心**：推理引擎 + 沙箱运行时 + Skill 注册表，保持精简。
- **Skill**：可插拔的能力扩展包，按需安装，沙箱内运行，不调用即静默。

### 4.1 Skill 的特点

| 特性 | 说明 |
|------|------|
| 沙箱执行 | Skill 运行在隔离沙箱内，无法访问核心数据与系统权限 |
| 按需激活 | 不主动调用时完全静默，不消耗资源 |
| 扫码安装 | 通过网站端 Skill 市场扫码即可安装到本地 |
| 可随时卸载 | 在设置中可查看已安装 Skill 并卸载 |

### 4.2 网站端 Skill 市场

访问 Memento 官方网站的 **Skill 市场** 页面，可浏览所有可用 Skill。

**安装流程**：

1. 在网站 Skill 市场中找到想要的 Skill，点击「安装到 Memento」。
2. 网站生成一个二维码。
3. 打开手机端 Memento → 设置 → Skill 管理 → 扫一扫。
4. 扫码后确认安装，Skill 包（ZIP 格式）下载并解压到本地沙箱目录。
5. 安装完成后，Skill 自动注册到核心，可在对话中按需调用。

### 4.3 以「漫剧 Skill」为例

**漫剧 Skill** 是 Memento Skill 市场的首发能力模块，本质上是一座完整的端侧漫剧加工厂。

#### 它做什么

漫剧 Skill 不是简单的"AI 画漫画"，而是一条完整的工业化创作工作流：

```
故事大纲 → 剧本拆解 → 角色设计 → 分镜脚本 → 逐帧渲染 → 排版合成 → 漫剧成片
```

| 环节 | Skill 工作 |
|------|-----------|
| 剧本拆解 | 将你的故事大纲拆解为可拍摄的场景与对白 |
| 角色设计 | 自动设定角色形象，并在所有分镜中保持一致 |
| 分镜脚本 | 规划每个画面的构图、机位、对白气泡 |
| 逐帧渲染 | 按分镜逐帧生成漫画风格画面（基于「理解→翻译→渲染」架构） |
| 排版合成 | 自动排版分格、添加对白气泡、生成封面与封底 |
| 输出 | 输出可分享的漫剧图集或长图 |

#### 为什么漫剧 Skill 是「理解→翻译→渲染」架构的最佳实践

漫剧加工厂的核心需求是**角色一致性**——同一个角色在所有分镜中必须长相一致。这正是 Memento 架构的强项：

- 像素生成模型：每个分镜的角色都可能变形
- Memento 架构：角色由 SVG 结构化描述定义，所有分镜复用同一描述，天然一致

#### 使用方式

安装后直接对话触发：

- 「帮我做一个关于小猫探险的 8 格漫剧」
- 「把这段故事做成漫剧：……」
- 「画一个赛博朋克风格的短篇漫剧，主角是侦探」

Skill 会自主完成全流程，过程中可随时介入调整（如「第 3 格换成特写」「对白改成中文」）。

#### 安装步骤

1. 访问网站 Skill 市场，找到「漫剧加工厂」。
2. 点击「安装」，用手机 Memento 扫描二维码。
3. 安装完成后，在对话中说出创作需求即可触发。
4. 不再需要时，前往 设置 → Skill 管理 → 漫剧加工厂 → 卸载。

> 💡 Skill 市场会持续更新，未来会有更多能力（绘画增强、文档处理、代码助手等）以 Skill 形式发布。

## 6. 常见问题

### Q: 下载模型时提示「403」或「下载中断」？

A: 检查网络环境。模型托管在阿里云 OSS，下载支持断点续传，重新进入应用会自动继续。

### Q: 引擎初始化时崩溃？

A: 通常是模型文件不完整或设备内存不足。请前往设置 → 卸载模型，重新下载。低端设备（< 6GB RAM）可能无法稳定运行。

### Q: 发送图片后提示「图片处理失败」？

A: 可能原因：
- 单张图片过大（建议 < 10MB）；
- 图片格式不被系统支持；
- 设备内存不足。建议选择较小的图片或减少一次发送的数量。

### Q: 视频输入后模型看不到内容？

A: 视频输入只采样前 **1 秒** 的 24 帧。若视频前 1 秒是黑屏或无关内容，模型理解会受限。建议：
- 拍摄/剪辑时让关键内容出现在前 1 秒；
- 或改用「多图输入」手动挑选关键帧。

### Q: 生成的图片是矢量风格，不是写实照片？

A: 这是**架构特性，不是缺陷**。Memento 采用「理解 → 翻译 → 渲染」架构，模型输出 SVG（矢量描述），渲染管线渲染为图片。优势是**一致性可控**和**结构可编辑**，代价是无法生成照片级写实。

- 需要写实照片？建议使用云端扩散模型。
- 需要一致性、可编辑、连续叙事？Memento 是正确选择。

### Q: 生成的视频很短（不到 1 秒）？

A: 视频时长 = 帧数 / 24fps。模型根据动画复杂度自由决定帧数（4-12 帧为宜），因此视频通常为 0.17s ~ 0.5s。这是端侧轻量级模型的 token 限制决定的。未来升级更大模型后可支持更长视频。

### Q: 编辑指令没生效？

A: 编辑模式需要满足两个条件：
1. 检测到编辑关键词（修改/替换/去掉/换成/改成/调整/edit/modify/replace/remove）；
2. 有最近一次生成的图片（应用会缓存其 SVG）。

若刚清空过对话或重启过应用，缓存的 SVG 可能已丢失，需要重新生成图片后再编辑。

### Q: 视频渲染失败？

A: 可能原因：
- 设备不支持 H.264 硬件编码（极少数老旧设备）；
- 模型输出的 SVG 包含不支持的元素；
- 存储空间不足。

建议重启应用后重试，或尝试更简单的动画描述。

### Q: Skill 安装后没有反应？

A: Skill 在沙箱中运行，只有当对话内容匹配其触发条件时才会激活。可尝试用更明确的指令触发（如「画一个漫剧」触发漫剧 Skill）。

### Q: 应用占用内存很大？

A: 模型加载后会常驻内存（约 600MB-1GB），这是端侧推理的必要开销。如需释放，可在设置中「卸载模型」，下次对话时会自动重新加载。
